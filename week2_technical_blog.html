Margin, Border, Padding Explained

Every HTML page is marked up by elements with corresponding tags.
Every element occupies a space on the page. The elements space can be given an outer buffer zone - a margin. This would specify what distance the designer wishes to set the element apart from its nearest element on the page. Much like a box, a margin can be thought of as having four sides. Top, Right, Bottom and Right. A margin amount can be given to any side independent of another side's amount.
The CSS selector can be left as margin and given a numeric value or a value of auto or inherit. Or the selector can target a specific side such as margin-top, margin-left, margin-bottom or margin-right. The values do not have to match and a designer can create a "box" varrying dimensions.
To see the margin seletor in action go here.


Getting back to the HTML element, its space can be delineated with a line of any given width, color and style, such as solid, dotted, dashed, double, groove, ridge, inset and outset. This border line is invisible by default on every element inside the HTML tag. Much like a box we draw on a page, a border has four sides. Top, Left, Bottom and Right. A border selector can be assigned to all four sides at once, or any of these sides individually. To see the border seletor in action go here.


The elements space can also be given a padded zone that goes between itself and its inner border, this zone is called - padding. Padding will effect the size of the overall element on the page, additional space will have to be accounted for it. It's internal to the element whereas margin is external to the element.
The CSS selector can be left as padding (targeting all 4 sides) or the selector can target a specific side such as margin-top, margin-left, margin-bottom or margin-right (similar to the way you wrote the margin selectors). The values do not have to match and a designer can create a box of varrying dimensions.
To see the padding in action go here.
